HTML5新特性总结

发表于 2018-12-12

HTML5新标签总结不包含已废除的标签

HTML5语义化标签：

<header>

定义 section 或 page 的顶部。主要用于放其所在区块的标题、简介，还可以放置搜索表单、logo图片等元素，根据最新标准还可以放<nav>导航

<nav>

定义导航链接。 只有语义化的作用，里面还需要加ul来构建导航


<article>

标签规定独立的自包含内容。可以独立于站点的其余部分进行分发，类似论坛帖子，报纸文章，用户评论等


<aside>

定义页面内容之外的内容。如文章或网站的侧栏（其他文章列表，广告）、文章的注解内容等。


<figure>

标签规定独立的流内容（图像、图表、照片、代码等等）。


<footer>

定义 section 或 page 的页脚。页脚通常包含文档的作者、版权信息、使用条款链接、联系信息等等。


<section>

是一个主题性的内容分组，通常用于对页面进行分块或者对文章等进行分段，只有元素内容会被列在文档大纲中时，才适合用section元素。section的作用是对页面上的内容进行分块，如各个有标题的版块、功能区或对文章进行分段，不要与有自己完整、独立内容的article混淆。例如一个网站有体育板块、国际板块、财经板块等等



文档级别标签:

<ruby> <rt> <rp>

ruby是一种排版注释系统，主要就有ruby、rt、rp三个标签，ruby 元素由一个或多个字符（需要一个解释/发音）和一个提供该信息的 rt 元素组成，还包括可选的 rp 元素，定义当浏览器不支持 “ruby” 元素时显示的内容。类似：

<ruby>
    厉<rt>li</rt>
    害<rt>hai</rt>
</ruby>

显示效果：
厉li害hai

<time>

标签定义公历的时间（24 小时制）或日期，时间和时区偏移是可选的。目前大多数浏览器不支持此标签,主要给搜索引擎使用。


<mark>

标签定义带有记号的文本。效果:mark


<wbr>

wbr元素用于让浏览器为长单词增加可选择的破折号，以便自动拆行。


<bdi>

标签允许您设置一段文本，使其脱离其父元素的文本方向设置。比如：Bill 80 把80放到标签里，可以通过标签的dir属性，来设置80放到Bill前还是Bill后。


表单：

email

输入邮箱地址，会在输入时提示要带@符号

<input type="email"/>

url

输入时输入的不是URL地址，会提示输入网址

<input type="url"/>

number

选取数字，可以设置最大、最小值，一次跳动大小。

<input type="number"/>

range

可以拖动的滑块，可是设置起始位置及一次拖动的变化量

<input type="range"/>

Date

时间表，用于选取时间

<input type="Date"/>

search

搜索框，和普通输入框不同的是最左边有个删除按键，可以一下清空搜索框。

<input type="search"/>

color

用调色板选取颜色。

<input type="color"/>

媒体标签：

<video>

标签定义视频，比如电影片段或其他视频流。有7个属性height（高）、width（宽）、loop（是否循环）、controls（如果出现该属性，则向用户显示控件，比如播放按钮。）、autoplay（如果出现该属性，则视频在就绪后马上播放。）、preload（如果出现该属性，则视频在页面加载时进行加载，并预备播放。如果使用 “autoplay”，则忽略该属性。）、src(要播放的视频的 URL。)

<audio>

标签定义声音，比如音乐或其他音频流。有5个属性autoplay(如果出现该属性，则音频在就绪后马上播放。部分游览器不兼容)、controls（如果出现该属性，则向用户显示控件，比如播放按钮。）、loop（如果出现该属性，则每当音频结束时重新开始播放。）、preload（如果出现该属性，则音频在页面加载时进行加载，并预备播放。如果使用 “autoplay”，则忽略该属性。）、src（要播放的音频的 URL。）

<embed>

标签定义嵌入的内容，比如插件。

其他标签:

<canvas>

标签定义图形，比如图表和其他图像，用javaScript绘制图像

<keygen>

标签规定用于表单的密钥对生成器字段。
当提交表单时，私钥存储在本地，公钥发送到服务器

<progress>

标签定义运行中的进度（进程）。可以使用标签来显示 JavaScript 中耗费时间的函数的进度。

<command>

标签可以定义命令按钮，比如单选按钮、复选框或按钮。
只有当 command 元素位于 menu 元素内时，该元素才是可见的。否则不会显示这个元素，但是可以用它规定键盘快捷键。

<deteils>

标签用于描述文档或文档某个部分的细节。

<dateilst>

标签定义选项列表。请与 input 元素配合使用该元素，来定义 input 可能的值。
